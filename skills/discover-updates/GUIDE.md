# Discover Updates - Detailed Guide

## Official Sources (Tool & Model Updates)

### 1. Anthropic Blog & News
- **URL**: https://www.anthropic.com/news
- **What to find**: Major model releases, API updates, research papers
- **Search for**: "Claude 3.x", "API", "features", release dates
- **Frequency**: Check monthly

### 2. Claude API Changelog
- **URL**: https://docs.anthropic.com/en/release-notes/api
- **What to find**: API changes, new parameters, deprecations
- **Search for**: Version numbers, breaking changes, new capabilities
- **Frequency**: Check before major projects

### 3. Claude Code CLI Releases
- **URL**: https://github.com/anthropics/claude-code/releases
- **What to find**: CLI features, bug fixes, new commands
- **Search for**: Version tags, feature additions, upgrade instructions
- **Frequency**: Check weekly

### 4. GitHub Discussions & Issues
- **URL**: https://github.com/anthropics/claude-code/discussions
- **What to find**: Feature requests, roadmap hints, official responses
- **Search for**: "roadmap", "planned", "upcoming", labels:enhancement
- **Frequency**: Check when planning new workflows

## Search Strategies

### Broad Discovery (Monthly)
1. Scan Anthropic blog for major announcements
2. Review last 3 GitHub releases
3. Check top posts on r/ClaudeAI (past month)
4. Search "Claude" on Hacker News (past month)

### Focused Research (When Planning Features)
1. Check API changelog for relevant capabilities
2. Search GitHub issues for similar use cases
3. Search Reddit for "how to [specific task]"
4. Look for community tools that solve similar problems

### Trend Monitoring (Weekly)
1. Subscribe to Claude Code CLI releases (watch repo)
2. Follow @AnthropicAI on Twitter/X
3. Quick scan r/ClaudeAI top posts (past week)

## Tips

### Efficient Searching
- Use site-specific search: `site:reddit.com/r/ClaudeAI workflow tips`
- Filter by date: Add "past month" or "past week" to searches
- Use GitHub advanced search: `is:issue label:enhancement`
- Check "trending" tabs on GitHub for community momentum

### Signal vs Noise
- **High signal**: Official releases, upvoted Reddit posts, active GitHub discussions
- **Low signal**: Single comments, unverified claims, outdated posts
- **Trust but verify**: Test claims before adopting

### Avoiding Overwhelm
- Focus on actionable updates (not just "nice to know")
- Prioritize official sources for accuracy
- Use community sources for inspiration, not gospel
- Timebox searches (30 min max for broad scan)

## Integration with Memory System

Save findings to:
- `~/.claude-memory/global/discoveries/[YYYY-MM-DD]-updates.md`
- Tag with: `#claude-updates`, `#tools`, `#best-practices`
- Reference in session logs when applying new techniques
